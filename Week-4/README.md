# E-commerce Sales Analysis Project<br>
<br>
# Project Overview<br>
This project demonstrates a full data analysis workflow for an e-commerce business using Python.<br>
It covers all the essential steps of data analysis: loading, cleaning, exploring, analyzing, visualizing, and reporting insights.<br>
<br>

Dataset Description:<br>
The dataset `sales_data.csv` contains sales records for an e-commerce store, including:<br>
- Date – Date of the transaction<br>
- Product – Product sold<br>
- Quantity – Number of units sold<br>
- Price – Price per unit<br>
- Total_Sales – Total sales amount for the transaction<br>
- Customer_ID – Unique identifier for each customer<br>
- Region – Customer’s region<br>
<br>

Project Goals:<br>
1. Data Cleaning and Preparation<br>
- Handle missing values and remove duplicate entries<br>
- Convert the Date column to datetime format for time-based analysis<br>

2. Exploratory Data Analysis<br>
- Explore dataset structure and summary statistics<br>
- Understand sales distribution across products and regions<br>

3. Sales Analysis<br>
;- Calculate total sales for the store<br>
- Identify the best-selling product<br>
- Compute average sales per product<br>
- Find top 3 products contributing to the revenue<br>

4. Data Visualization<br>
- Bar chart: Total sales by product<br>
- Pie chart: Sales distribution by product<br>
- Line chart: Monthly sales trend<br>

5. Reporting and Insights<br>
- Save all charts in the visuals/ folder<br>
- Provide actionable insights based on sales trends<br>
- Prepare a professional report in the reports/ folder<br>
<br>

Why This Project Matters:<br>
- Helps understand customer preferences and product performance<br>
- Provides insights into sales trends over time<br>
- Enables data-driven decisions for inventory, marketing, and sales strategies<br>
- Demonstrates a complete Python-based data analysis workflow, suitable for academic or professional projects<br>
